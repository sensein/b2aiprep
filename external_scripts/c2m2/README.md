# Creating C2M2 documentation

Using the following [guide](https://github.com/MaayanLab/cfde-c2m2), create the folder that you want to generate C2M2 in, then run:

```cfde-c2m2 init```

## From Physionet bundles run the following

1. Fill the subject file folders for peds and adults

- ```python fill_subject_files.py /adult/bundle/ /path/to/c2m2_output/```
- ```python fill_subject_files.py /peds/bundle/ /path/to/c2m2_output/ --peds```

2. Fill in files that use the bundles for listing files and creating connections between subjects (mostly removed), biosamples, diseases, and files

- ```python bundle_to_c2m2.py /adult/bundle/ /path/to/c2m2_output/ --c2m2_id_map /path/to/id_map.csv```
- `python bundle_to_c2m2.py /peds/bundle/ /path/to/c2m2_output/ --c2m2_id_map /path/to/id_map.csv --peds`

3. Copy static files
- ```python finalize.py /path/to/c2m2_output/```

## Finalize the C2M2 packaging

The following should all be ran within the C2M2 path that was created.

Run: `cfde-c2m2 prepare`

Run: `cfde-c2m2 validate`

Fix whatever errors appear in validation (hopefully none) and finalize the package with `cfde-c2m2 package -o /output/path/for/zip/2026_03_C2M2.zip`

# Post Metadata Generation

- Upload to [data.cfde.cloud](data.cfde.cloud)
- Make sure the file generated by `--c2m2_id_map` is kept in a safe place for future runs
